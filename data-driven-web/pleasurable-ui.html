Sprint 11: Pleasurable UI

Topics

  • 13/05/2024

In de sprint Pleasurable UI leer je hoe je interfaces kan coderen waar gebruikers blij van worden.

The hierarchy of user needs defines basic user needs that interfaces must fulfill before more advanced needs can be addressed such as pleasurable. - Therese Fessenden

De focus van deze sprint ligt op samenwerken, client-side scripting en creative coding.

Gedragscriteria

  • M: Je combineert aangeboden principes en conventies op het gebied van frontend design en development (2.1.2)
  • S: je werkt in teams, vraagt begeleiding waar nodig en geeft feedback aan teamleden (2.2.1)
  • S: Je draagt verantwoording voor eigen resultaten, verwerkt ontvangen feedback en wijst teamleden op hun verantwoording (2.2.2)
  • C: Je kan ontwerpkeuzes, eigen ideeën en producten begrijpelijk overbrengen aan belanghebbenden (2.3.2)
  • P: Je bedenkt en implementeert complexere frontend code en gebruikt daarbij API's en databases (2.4.3)
  • L: Je maakt aangeboden en zelf gevonden materie eigen en gebruikt dit bij leertaken, deelt ervaring binnen de squad (2.5.2)
